Write an 8085 program that reads two 8-bit numbers from memory (e.g., `2500H` and `2501H`), multiplies them by **repeated addition**, and stores the 16-bit product across two bytes in memory (e.g., `3000H`-`3001H`).

This approach is straightforward and avoids indirect multiplication or shift-based algorithms, making it easy to follow and educational.

## ‚ö° TL;DR ‚Äî Final Working Code

```asm
        ; Interface
        FIRST_NUM_ADDR   EQU 2500H
        SECOND_NUM_ADDR  EQU 2501H
        RESULT_ADDR      EQU 3000H

        ; Step 1: Load inputs
        LXI H, FIRST_NUM_ADDR
        MOV B, M             ; B ‚Üê multiplicand
        INX H
        MOV C, M             ; C ‚Üê multiplier

        ; Step 2: Initialize result
        LXI H, 0000H         ; HL ‚Üê result (16-bit accumulator)

        ; Step 3: Multiply using repeated addition
MUL_LOOP:
        MOV A, L
        ADD B
        MOV L, A
        JNC SKIP_INC
        INR H                ; Carry to high byte

SKIP_INC:
        DCR C
        JNZ MUL_LOOP

        ; Step 4: Store result
        LXI D, RESULT_ADDR
        MOV M, L             ; Store low byte
        INX D
        MOV M, H             ; Store high byte
        HLT
```

## üß± Step 2: Initialize Result Registers

We‚Äôll now prepare registers to **hold the result** of multiplication.

Since multiplying two 8-bit numbers can produce a 16-bit result (maximum `255 √ó 255 = 65025` = `FE01H`), we need:

* One register for the **low byte** (e.g., `L`)
* One for the **high byte** (e.g., `H`)
* An additional register (`C`) will act as a loop counter (already holds one multiplier)

---

### üß† What We're Doing

* Clear the result registers (HL pair) to start from zero
* Use `C` as the counter to add the multiplicand (`B`) repeatedly

### üßæ Code (Initialize Result)

```asm
                 ; B = multiplicand, C = multiplier
```

Now we‚Äôre ready to perform the actual multiplication in the next step using **repeated addition with carry** into HL.

## üß± Step 3: Add the Multiplicand Repeatedly

Now we‚Äôll perform the multiplication using **repeated addition**.

Since register `C` holds the multiplier, we‚Äôll add the multiplicand (`B`) to the result (`HL`) that many times. To properly handle 16-bit accumulation, we‚Äôll add `B` to the **L register**, and propagate any carry into **H**.

---

### üß† What We're Doing

* Loop `C` times:

  * Add `B` to `L` (low byte)
  * If there‚Äôs a carry, increment `H` (high byte)
* Decrement `C` until zero

---

### üßæ Code (Multiplication Loop)

```asm
MUL_LOOP:
    MOV A, L       ; A ‚Üê current low byte
    ADD B          ; A ‚Üê A + multiplicand
    MOV L, A       ; Store updated low byte

    JNC SKIP_INC   ; If no carry, skip
    INR H          ; Else, increment high byte

SKIP_INC:
    DCR C          ; Decrement loop counter
    JNZ MUL_LOOP   ; Repeat if C ‚â† 0
```

## üß± Step 4: Store the 16-bit Result in Memory

After computing the result in the HL register pair:

* `L` holds the **lower byte** of the product
* `H` holds the **upper byte** (in case of overflow)

Now we need to store both bytes at predefined memory addresses for output.

---

### üì¶ Output Mapping

| Address | Purpose              |
| ------- | -------------------- |
| `3000H` | Low byte of product  |
| `3001H` | High byte of product |

